# 公司督办管理系统 - AI 开发规范

## 一、项目定位
本系统用于管理公司重点工作、主要工作、待办事项，核心是事项、节点、审批、权限、统计、提醒闭环。

## 二、技术栈
- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- shadcn/ui
- PostgreSQL
- Prisma

## 三、核心业务原则
1. 权限优先于页面展示。
2. 状态流转必须由 workflow 层统一处理，页面不得直接改状态。
3. 统计口径必须和列表口径一致。
4. 待我处理 = 待我审批 + 待我办理。
5. 待审批只统计当前用户作为 currentApprover 的审批事项。
6. 待处理统计当前用户作为责任人、责任部门领导、待分解/待完成/待调整处理人的事项。
7. 重点工作、主要工作、待办事项可以有不同流程，但命名和统计口径必须统一。

## 四、开发边界
- 不要擅自重构数据库结构，除非 Issue 明确要求。
- 不要擅自引入大型状态管理或流程引擎。
- 不要改变既有角色枚举。
- 不要把测试账号密码硬编码到前端页面。
- 不要删除历史审批记录。
- 不要绕过服务端权限校验。

## 五、每次修改必须完成
1. TypeScript 检查通过。
2. npm run build 通过。
3. 涉及权限时补充测试账号验证说明。
4. 涉及统计时说明统计 SQL / 服务函数口径。
5. 涉及审批时说明状态流转前后变化。
