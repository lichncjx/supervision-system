# 公司督办管理系统 - 项目说明

本文档说明项目背景、模块、目录、数据模型和 API 概览。权限、审批流、状态机、统计口径以 `docs/业务规则.md` 为准；部署和迁移以 `docs/部署说明-群晖.md` 为准。

## 项目概览

公司督办管理系统用于跟踪和管理公司重点工作、主要工作、待办事项。系统围绕事项创建、审批流转、节点分解、见证材料、完成确认、统计看板和发布验收形成闭环。

## 技术栈

- Next.js 15 App Router
- React 19
- TypeScript 5
- Tailwind CSS 4
- shadcn/ui
- PostgreSQL
- Prisma
- pnpm

## 功能模块

### 登录与认证

- 用户名和密码登录。
- 会话管理。
- 当前用户信息接口。
- 修改密码。

### 首页 Dashboard

- 总事项、待审批、待办理、待我处理、进行中、已完成、超期等统计。
- 重点工作、主要工作、待办事项分类统计。
- 最近事项和待处理入口。
- 统计口径必须与列表查询口径一致。

### 事项管理

- 重点工作：公司级重要事项。
- 主要工作：部门主要工作。
- 待办事项：日常任务或领导交办事项。
- 支持列表筛选、搜索、详情查看、节点维护、见证材料、审批记录。
- 支持 Excel 导入导出。

### 审批中心

- 待我处理列表。
- 审批通过、退回。
- 调整、取消、完成确认等流程操作。
- 审批人和状态流转必须由服务端校验。

### 系统管理

- 部门管理。
- 用户管理。
- 角色和权限说明。
- 操作日志。

### 角色语义说明

`DEPARTMENT_MANAGER` 表示系统办理人、部门事项管理岗 / 部门督办管理员，负责代表部门填报、办理、提交材料，不等于事项里的实际责任人。重点工作、主要工作的事项实际责任人目标上使用 `responsiblePerson` 姓名文本表达。

## 数据模型概览

### Department

部门表，包含部门名称、编码、是否业务部门等信息。默认部门由 `prisma/seed.ts` 初始化。

### User

用户表，包含用户名、密码哈希、姓名、角色、所属部门、启用状态等信息。

### WorkItem

事项表，核心字段包括：

- 事项类型：`type`
- 事项状态：`status`
- 当前动作：`action`
- 基本信息：`title`、`businessCategory`、`workItem`、`workNode`、`completeTime`、`completeForm`、`isInnovation`
- 部门关联：
  - `departmentId` — 重点/主要工作的单一主办部门
  - `responsibleDepartmentIds` — 目标字段，待办事项主责部门 ID 数组
  - `departmentIds` — 当前字段，若暂不迁移则作为 `responsibleDepartmentIds` 的兼容读取来源
  - `cooperateDepartmentIds` — 待办事项配合部门 ID 数组
- 系统用户 / 流程字段（用于权限、流程、审批）：
  - `creatorId` — 创建人
  - `firstSubmitterId` — 首次提交审批人
  - `proposedLeaderId` — 提出领导
  - `approvalLeaderId` — 审批领导（默认等于 proposedLeaderId）
  - `currentApproverId` / `currentApproverRole` — 当前审批人
- 目标业务责任人姓名字段（仅用于展示/导出/留痕，不参与权限）：
  - `responsibleLeader` — 事项业务责任领导姓名文本，不是 User 外键，不是 Member 外键
  - `responsiblePerson` — 事项实际责任人姓名文本，不是 User 外键，不是 Member 外键；当前 schema 尚需迁移或由兼容字段派生
  - `responsiblePersons` — 待办事项主责责任人姓名文本或 JSON 数组
  - `cooperatePersons` — 待办事项配合责任人姓名文本或 JSON 数组
- 当前实现兼容字段：
  - `deptLeaderId` / `deptLeaderName` — 当前实现中用于部门领导用户和姓名快照，不等同于目标业务责任领导
  - `deptManagerId` / `deptManagerName` — 当前实现中用于部门事项管理岗用户和姓名快照，不等同于事项实际责任人
  - `supervisor` — 历史文本字段，不作为目标责任字段，与 `SUPERVISOR` 角色无关
- 待办事项专用：`proposedScene`、`formedTime`、`workPlan`、`planCompleteTime`、`progress`
- 证明材料/调整/取消/退回：`proof`、`adjustReason`、`cancelReason`、`rejectReason`、`rejectedFromStatus`
- JSON 扩展：`nodes`、`adjustHistory`
- 附件关联：`attachments`（一对多）

### WorkflowRecord

流程记录表，记录每次流程动作的发起人、审批人、流转前状态、流转后状态、意见和时间。审批记录不得删除。

### Attachment

附件表，记录上传文件与事项、用户的关联。核心字段包括：

- `workItemId` — 关联事项
- `userId` — 上传用户
- `fileName`、`filePath`、`fileSize`、`fileType` — 文件基本信息
- `uploadedAt` — 上传时间
- `category` — 附件分类：`"general"`（普通附件，默认值）或 `"evidence"`（证明材料/见证材料）

### OperationLog

操作日志表，记录用户在系统中的关键操作。

## API 概览

### 认证

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/login` | POST | 登录 |
| `/api/auth/logout` | POST | 登出 |
| `/api/auth/me` | GET | 当前用户 |
| `/api/auth/change-password` | POST | 修改密码 |

### 部门、用户、角色

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/departments` | GET | 部门列表 |
| `/api/users` | GET/POST | 用户列表、创建用户 |
| `/api/users/[id]` | GET/PUT/DELETE | 用户详情、更新、删除 |
| `/api/users/[id]/password` | PUT | 重置用户密码 |
| `/api/users/[id]/status` | PUT | 修改用户状态 |
| `/api/users/by-department` | GET | 按部门获取用户 |
| `/api/users/department-managers` | GET | 部门事项管理员列表 |
| `/api/users/department-leaders` | GET | 部门领导列表 |
| `/api/users/company-leaders` | GET | 公司领导列表 |
| `/api/roles` | GET | 角色列表 |

### 事项和流程

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/works` | GET/POST | 事项列表、创建事项 |
| `/api/works/[id]` | GET/PUT/DELETE | 事项详情、更新、删除 |
| `/api/works/[id]/workflow` | GET/POST | 流程记录、流程操作 |

### 附件和上传

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/upload` | POST | 文件上传 |
| `/api/attachments/[id]` | DELETE | 删除附件 |
| `/api/attachments/[id]/download` | GET | 下载附件 |

### Excel、统计和日志

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/excel/export` | GET | 导出事项 |
| `/api/excel/import/[type]` | POST | 导入事项 |
| `/api/excel/template/[type]` | GET | 下载导入模板 |
| `/api/excel/completion-rate` | GET | 导出完成率统计 |
| `/api/dashboard/summary` | GET | 首页统计 |
| `/api/dashboard/completion-rate` | GET | 完成率统计 |
| `/api/operation-logs` | GET | 操作日志 |
| `/api/health` | GET | 健康检查 |
