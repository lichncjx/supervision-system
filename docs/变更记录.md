# 变更记录

本文档记录业务规则、状态机、权限、统计和数据设计相关的重要文档变更。

## 2026-05-09

### 第二阶段：统一服务端权限与数据范围

1. 完善 `src/lib/server-permissions.ts`，集中提供事项查看、审批、办理、编辑、附件查看/上传/删除、可见范围查询构造、主责部门和配合部门读取 helper。
2. `GET /api/works` 改为使用 `buildWorkVisibilityWhere(user)`，不再把 `VICE_PRESIDENT` / `PRESIDENT` 默认当作全局可见角色。
3. `GET /api/works/[id]`、`PUT /api/works/[id]`、`GET /api/works/[id]/workflow` 接入统一查看/编辑权限。
4. `GET /api/dashboard/summary` 移除接口内部分散的 canApprove/canHandle 判断，改为统一权限 helper；本阶段不重构为目标 `GET /api/dashboard`。
5. `GET /api/dashboard/completion-rate` 与 `GET /api/excel/completion-rate` 改为通过 `getResponsibleDepartmentIds(work)` 按主责部门集合统计，配合部门不计入完成率。
6. `src/lib/attachment-permissions.ts` 改为委托统一服务端权限模块，并让上传、下载、删除附件接口传入多主责部门、配合部门和当前审批字段。
7. 本阶段不修改 Prisma schema、不做 9 状态迁移、不修改 workflow 状态流转。

### 第二阶段收口：target-contract 预期调整

1. 将第二阶段已修复的 9 个 target-contract expected failure 调整为正常 `PASS`：`/api/works` 可见范围、`VICE_PRESIDENT` 默认不可见其他副总事项、部门多主责可见、配合部门可见、完成率 B 部门多主责统计。
2. 已接入统一服务端权限 helper 的接口包括：`GET /api/works`、`GET /api/works/[id]`、`PUT /api/works/[id]`、`GET /api/works/[id]/workflow`、`GET /api/dashboard/summary`、`GET /api/dashboard/completion-rate`、`GET /api/excel/completion-rate`、附件上传/下载/删除接口。
3. 保留 `/api/dashboard/summary` 旧状态分组、旧统计口径、临期口径，以及目标 `GET /api/dashboard`、9 状态迁移、Excel 普通导入导出、workflow POST 状态流转为后续阶段范围。
4. 本次收口只调整 target-contract 预期和文档记录，不继续扩大业务改造范围，不修改 Prisma schema。

### 第一阶段：统一状态元数据

1. 新增 `src/lib/work-status.ts`，集中维护当前 15 个 `WorkItemStatus` 的状态值、Prisma 值、中文文案、视觉分组、Badge 样式、终态/审批中/办理中/进行中/临期超期/legacy 标记。
2. 页面状态 Badge、列表状态强调色、前端事项 store 的状态文案和临期/超期排除判断、Excel 导出状态文案改为读取统一状态元数据。
3. 保留 `pending_approval`、`active`、`decomposed` 作为历史展示兼容状态，不纳入当前 15 状态集合，不扩展业务语义。
4. 本阶段不修改数据库 schema、不做 9 状态迁移、不调整 workflow 状态流转、不改变 Dashboard 统计口径。

## 2026-05-09 口径修正

1. 重排 `docs/业务规则.md`，明确分为“目标业务规则 / 当前实现差异 / 迁移前兼容说明”，避免 15 状态当前实现和 9 状态目标架构混写。
2. 修正重点/主要事项责任字段口径：`responsibleLeader` / `responsiblePerson` 是姓名文本，不是 `User` 外键，不是 `Member` 外键，不参与权限判断。
3. 修正公司领导查看权限口径：`ADMIN` / `SUPERVISOR` 全局查看，`VICE_PRESIDENT` / `PRESIDENT` 不默认全局。
4. 修正 Dashboard 轻量列表字段名，使用当前真实字段 `completeTime` / `planCompleteTime`，不再把 `planEndDate` 写成当前数据库字段。
5. 补充 `responsibleDepartmentIds` 落地兼容规则：迁移时重命名 `departmentIds`，不迁移时统一通过 `getResponsibleDepartmentIds(work)` 读取。
6. 再次确认本轮不新增 `Member` / `DepartmentMember` / `WorkParticipant`，业务责任人字段仅用于展示、导入导出和历史留痕。

## 2026-05-09 目标口径自动化验证

4. 新增 `tests/target-contract/target-contract-data.cjs`，集中维护目标测试数据和目标口径计算规则。
5. 新增 `docs/待修复问题.md`，记录当前接口与目标口径之间的 expected failure。
6. 更新 `docs/回归测试清单.md`，将策略调整为目标口径自动化测试优先，不再固化当前错误统计口径。

## 2026-05-09 目标口径脚本安全加固

1. 新增 target-contract 脚本安全检查，要求 `TARGET_CONTRACT_ENV=local|test` 和 `TARGET_CONTRACT_RESET=1`。
2. seed / verify 脚本会输出脱敏后的 `DATABASE_URL`。
3. seed / verify 脚本拒绝 Vercel 环境、Supabase/Vercel/production/preview 字样数据库 URL，以及非本地主机数据库。
4. verify 脚本增加 base-url 本地限制和请求超时。
5. 更新相关文档中的推荐运行命令和安全说明。
