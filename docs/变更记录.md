### 第六阶段 PR 6.2：workflow 新状态流转与审批权限同步

1. `POST /api/works/[id]/workflow` 已按 9 状态状态机重构：`DRAFT -> PROPOSING -> IN_PROGRESS`、`PENDING_DECOMPOSE -> PROPOSING -> IN_PROGRESS`、`IN_PROGRESS -> ADJUSTING / CANCELLING / COMPLETING`、草稿取消 `DRAFT -> CANCELLED`。
2. 进入审批态时统一写入 `beforeApprovalStatus` 和 `approvalType`；审批仍有下一节点时状态保持审批态，仅切换 `currentApproverId` / `currentApproverRole`；审批结束或退回时清空审批辅助字段。
3. 审批结束目标状态按 `approvalType` 决定：`PROPOSE` / `ADJUST` -> `IN_PROGRESS`，`CANCEL` -> `CANCELLED`，`COMPLETE` -> `COMPLETED`。
4. 审批退回统一恢复到 `beforeApprovalStatus`，因此普通立项退回回 `DRAFT`，公司领导发起待办分解退回回 `PENDING_DECOMPOSE`，调整 / 取消 / 完成退回回 `IN_PROGRESS`。
5. 重点工作取消主要领导审批不再使用 `PENDING_MAIN_LEADER_CANCEL`，而是在 `CANCELLING` 下切换主要领导审批节点。
6. `src/lib/server-permissions.ts` 中 `canApproveWorkItem` / `canHandleWorkItem` 已同步新审批态和办理态；审批优先按 `currentApproverId` 精确匹配，`currentApproverRole` 作为节点辅助。
7. target-contract 新增 workflow POST 路径断言；本次运行结果为 `PASS 101`，无 `EXPECTED_FAILURE`、`UNEXPECTED_PASS` 或普通 `FAIL`。
8. 页面和接口状态筛选完整适配、退回待修改展示、legacy 回归脚本清理仍留到 PR 6.3 / PR 6.4。
