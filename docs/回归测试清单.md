## 一、新建事项

### 1.1 新建重点工作

- [ ] **角色**：`dept_manager`（综合处）
- [ ] 打开 `/priority/new`
- [ ] 填写业务类别、工作事项、是否为创新工作（是/否）、节点、节点完成时间
- [ ] 选择责任部门（单选下拉）
- [ ] 选择部门领导（下拉仅显示所选部门的 `DEPARTMENT_LEADER` 角色用户）
- [ ] 选择主管人员（下拉仅显示所选部门的 `DEPARTMENT_MANAGER` 角色用户）
- [ ] 提交 → 状态变为 `pending_dept`
- [ ] 详情页显示正确的部门领导姓名和主管人员姓名

### 1.2 新建主要工作

- [ ] **角色**：`dept_manager`（综合处）
- [ ] 打开 `/main/new`
- [ ] 填写业务类别、工作事项、节点、节点完成时间
- [ ] 选择责任部门、部门领导、主管人员
- [ ] 提交 → 状态变为 `pending_dept`
- [ ] 详情页显示正确

### 1.3 新建待办事项

- [ ] **角色**：`dept_manager`（综合处）
- [ ] 打开 `/todo/new`
- [ ] 选择事项提出领导（`VICE_PRESIDENT` / `PRESIDENT`）
- [ ] 填写提出场景、待办事项、形成时间
- [ ] **主责部门**：MultiSearchSelect 可多选（如 XD/ZL），搜索可用
- [ ] **主责责任人**：下拉候选来自已选主责部门的人员，选中后显示 badge
- [ ] **配合部门**：MultiSearchSelect 可多选
- [ ] **配合责任人**：下拉候选来自已选配合部门的人员
- [ ] **删除主责部门**（取消勾选 XD）→ 仅 XD 独有的责任人被清空，ZL 仍有的同名者保留
- [ ] **删除配合部门**→ 同理过滤配合责任人
- [ ] 填写任务节点（至少 1 个）、节点完成时间
- [ ] 提交 → 状态按角色流转
- [ ] 详情页主责部门显示多部门名称（`/` 分隔或 map 拼接）
- [ ] 详情页主责责任人显示姓名（`、` 拼接）
- [ ] 详情页配合部门、配合责任人显示正确

### 1.4 公司领导新建待办事项

- [ ] **角色**：`vice_president`
- [ ] 打开 `/todo/new`，事项提出领导**自动填为本人的 id 且不可修改**
- [ ] 选主责部门、主责责任人
- [ ] 提交 → 状态进入 `pending_decompose`（待部门分解）
- [ ] `firstSubmitterId` 不在此步骤写入

## 二、审批流程

### 2.1 部门领导审批

- [ ] **角色**：`dept_leader`（综合处）
- [ ] 在审批中心或事项详情看到待审批事项（`pending_dept`）
- [ ] 点击"审批通过" → 状态进入 `pending_company`
- [ ] 点击"退回" → 填写退回原因 → 状态变为 `rejected`，退回给 `firstSubmitterId`

### 2.2 公司领导审批

- [ ] **角色**：`vice_president`
- [ ] 在审批中心看到待审批事项（`pending_company`）
- [ ] 点击"审批通过" → 状态进入 `approved` 或 `in_progress`
- [ ] 点击"退回" → 退回给 `firstSubmitterId`

### 2.3 待办事项分解审批

- [ ] **角色**：`dept_manager`（综合处）
- [ ] 公司领导创建的待办事项处于 `pending_decompose`
- [ ] 进入详情页，看到分解面板
- [ ] 填写工作计划、计划完成时间、任务节点、节点完成时间
- [ ] 提交分解 → 进入 `pending_dept`

## 三、退回处理

### 3.1 firstSubmitter 重新提交

- [ ] **前置**：事项被退回，`status = rejected`
- [ ] **角色**：仅 `firstSubmitterId` 对应用户（fallback `creatorId`）
- [ ] 在待我处理列表看到该退回事项
- [ ] 点击进入详情 → 看到退回原因（红色块）
- [ ] 看到退回处理面板（WorkReturnedPanel）
- [ ] 点击"修改后重新提交" → 编辑表单出现
- [ ] **如果是 TODO**：主责部门、主责责任人、配合部门、配合责任人使用 MultiSearchSelect 可正常回显和编辑
- [ ] 修改后填写"修改说明" → 提交
- [ ] 状态重新进入正式审批链路，`firstSubmitterId` 不覆盖
- [ ] 生成新的 workflow record

### 3.2 非 firstSubmitter 不可见退回处理面板

- [ ] **角色**：同部门其他 `dept_manager` / `dept_leader`（非 `firstSubmitterId`）
- [ ] 进入该退回事项详情页 → **不显示**退回处理面板
- [ ] 公司领导创建的待办退回后 → 公司领导的待我处理列表中**不出现**

## 四、附件与证明材料

### 4.1 普通附件上传和删除

- [ ] **角色**：`dept_manager`（同部门，事项 `status` 非 `completed` / `cancelled` / `rejected`）
- [ ] 在详情页普通附件面板上传文件
- [ ] 文件出现在附件列表中，可下载
- [ ] 上传者本人可以删除自己上传的附件
- [ ] 同部门其他用户（非上传者）**不可删除**该附件
- [ ] `SUPERVISOR` / `ADMIN` 可删除所有附件

### 4.2 evidence 附件上传和删除

- [ ] **角色**：有上传权限的用户
- [ ] 在操作面板选择 evidence 文件上传
- [ ] 文件出现在见证材料区（非普通附件区）
- [ ] 普通附件面板**不显示** evidence 附件
- [ ] 删除权限与普通附件一致

### 4.3 deptManager 上传权限

- [ ] **角色**：事项 `deptManagerId` 对应用户（重点/主要工作）
- [ ] 在 `rejected` 状态下可上传普通附件和证明材料（退回后补充材料）
- [ ] 在 `completed` / `cancelled` 状态下不可上传

### 4.4 配合部门附件权限

- [ ] **角色**：配合部门（`cooperateDepartmentIds`）的用户
- [ ] 可查看和下载附件
- [ ] **不可上传**附件（不因配合部门身份获得上传权限）
- [ ] **不可删除**他人附件

## 五、Excel 导入导出

### 5.1 模板下载

- [ ] 在 `/priority`、`/main`、`/todo` 页面点击"下载模板"
- [ ] 下载成功，文件打开正常
- [ ] TODO 模板包含示例行，部门列使用 code 格式（如 `XD/ZL`）
- [ ] Priority/Main 模板包含示例行

### 5.2 导入重点工作

- [ ] 按模板填写数据，责任部门填 code（如 `JH`）或全名（如 `计划生产处`）
- [ ] 上传 → 导入成功，生成 draft 状态事项
- [ ] 责任部门错误时有明确提示

### 5.3 导入主要工作

- [ ] 按模板填写 → 上传 → 成功
- [ ] 部门匹配同时支持 name 和 code

### 5.4 导入待办事项

- [ ] 按模板填写，主责部门用 `/` 分隔（如 `XD/ZL`）
- [ ] 配合部门用 `/` 分隔（如 `JH`）
- [ ] 主责责任人和配合责任人也用 `/` 分隔
- [ ] 上传成功 → 检查详情页：
  - `departmentIds` = `[XD的id, ZL的id]`
  - `cooperateDepartmentIds` = `[JH的id]`（**不再为空**）

- [ ] 主责部门名不存在 → 报错并阻止导入
- [ ] 配合部门名不存在 → 报错并阻止导入

### 5.5 导出

- [ ] 导出全量或按类型导出
- [ ] 部门列显示 code（非全名）

## 八、权限边界（综合检查）

### 8.1 可见性

- [ ] 部门角色只能看到本部门 + 配合部门的事项
- [ ] 公司级角色（`ADMIN` / `SUPERVISOR` / `VICE_PRESIDENT` / `PRESIDENT`）可看全部
- [ ] 首页隐藏按钮不等于权限，关键操作需服务端校验

### 8.2 编辑权限

- [ ] 只能修改 `draft` 或 `rejected` 状态的事项
- [ ] 部门角色只能修改本部门事项
- [ ] `ADMIN` / `SUPERVISOR` 可修改所有

### 8.3 删除权限

- [ ] 只有 `ADMIN` 可以删除事项
- [ ] 退回事项可由 `firstSubmitterId` 用户在退回面板中删除
