# 回归测试清单

本文记录当前系统发布前应执行的回归验证。当前运行态为 9 状态 + Phase 8 责任方模型。

## 当前 9 状态

1. `DRAFT`
2. `PENDING_DECOMPOSE`
3. `PROPOSING`
4. `IN_PROGRESS`
5. `ADJUSTING`
6. `CANCELLING`
7. `COMPLETING`
8. `COMPLETED`
9. `CANCELLED`

关键口径：

1. `REJECTED` 已取消；退回待修改 = `DRAFT` + 退回痕迹。
2. `APPROVED` 已取消；立项通过后进入 `IN_PROGRESS`。
3. `responsibleDepartmentIds` / `cooperateDepartmentIds` / `responsiblePersons` / `cooperatePersons` 已删除（Phase 8B）。
4. 责任方模型：`departmentId`（唯一主责部门）+ `responsibleLeader` + `responsiblePerson` + `cooperators`（Phase 8B/C/D）。
5. 完成率只按 `departmentId`；cooperators 不计入。
6. 姓名文本字段不参与权限。
3. 完成审批统一为 `COMPLETING`。
4. 取消审批统一为 `CANCELLING`。
5. `PENDING_DECOMPOSE` 仅表示公司领导发起待办后的待分解业务状态。
6. `approvalType` 表示审批类型，`beforeApprovalStatus` 表示审批退回路径。

## target-contract 覆盖重点

1. 角色登录与服务端权限范围。
2. `/api/works` 可见范围、状态筛选和派生筛选。
3. `/api/dashboard/summary` 和 `GET /api/dashboard` 统计口径一致。
4. 首页轻量列表临期 / 超期 / 待处理口径。
5. workflow 9 状态流转：
   - `DRAFT -> PROPOSING -> IN_PROGRESS`
   - `PENDING_DECOMPOSE -> PROPOSING -> IN_PROGRESS`
   - `PROPOSING` 退回到 `beforeApprovalStatus`
   - `IN_PROGRESS -> ADJUSTING -> IN_PROGRESS`
   - `IN_PROGRESS -> CANCELLING -> CANCELLED`
   - `IN_PROGRESS -> COMPLETING -> COMPLETED`
   - `DRAFT -> CANCELLED`
6. `beforeApprovalStatus` / `approvalType` 写入、保持和清空。
7. 普通 Excel 导入导出权限、字段和状态口径。
8. 旧状态筛选和旧状态导入负向断言。

## 手动回归建议

### 页面状态筛选

1. 类型列表页只展示当前 9 状态和派生筛选项。
2. “退回待修改”显示为草稿的派生状态，不写入数据库状态。
3. `/status/inProgress` 只包含 `IN_PROGRESS`。
4. `/status/approving` 覆盖 `PROPOSING`、`ADJUSTING`、`CANCELLING`、`COMPLETING`。
5. `/status/handling` 不依赖旧状态，退回待修改仅归属 `firstSubmitterId ?? creatorId`。
6. 临期 / 超期排除 `COMPLETED`、`CANCELLED`。

### workflow

1. 普通事项提交、审批、退回、重新提交。
2. 公司领导发起待办、部门分解、审批、退回到待分解。
3. 调整、取消、完成审批通过和退回。
4. 重点工作取消的主要领导节点保持在 `CANCELLING`，最终通过后进入 `CANCELLED`。
5. 草稿取消直接进入 `CANCELLED`。

### Excel

1. 普通导入默认 `DRAFT`。
2. 普通导入不允许直接导入审批态、终态或旧状态。
3. 普通导出状态文案来自统一状态元数据。
4. 普通导出范围与 `/api/works` 可见范围一致。

### Dashboard

1. 首页不再通过 `/api/works` 拉全量事项后前端计算待处理和临超期列表。
2. `GET /api/dashboard` 返回 `summary` 与轻量列表。
3. `GET /api/dashboard/summary` 仍保留兼容。
4. 完成率接口当前仍可独立调用。
