# 部署说明

本文补充 Vercel / Supabase 测试环境初始化流程。当前 Vercel production / preview 均按测试环境使用，不承载正式生产数据，因此允许清理业务测试数据后重新注入演示数据。

## Vercel / Supabase 测试环境初始化

### 1. 配置环境变量

在 Vercel 项目环境变量中配置：

```bash
DATABASE_URL=postgresql://...
DIRECT_URL=postgresql://...
```

`DATABASE_URL` 用于应用运行和 Prisma Client 连接；`DIRECT_URL` 用于 Prisma migration 直连 Supabase 数据库。具体连接串以 Supabase 项目提供的信息为准。

### 2. 执行数据库迁移

在本地或 CI 环境中使用远程测试库连接串执行：

```bash
npx prisma migrate deploy
```

`migrate deploy` 只应用尚未执行的 Prisma migrations，不会自动清理业务数据，也不会注入演示数据。

### 3. 注入测试演示数据

确认当前 `DATABASE_URL` 指向远程测试库后执行：

```bash
PREVIEW_SEED=1 PREVIEW_SEED_PASSWORD=123456 node scripts/seed-preview-data.cjs
```

Windows PowerShell 可使用：

```powershell
$env:PREVIEW_SEED='1'
$env:PREVIEW_SEED_PASSWORD='123456'
node scripts/seed-preview-data.cjs
```

脚本会清理业务表数据，但不会删除 `_prisma_migrations`，不会 drop database。清理顺序为：

```text
attachments -> workflow_records -> operation_logs -> work_items -> users -> departments
```

随后脚本会创建基础部门、测试用户和 9 状态演示事项。事项标题统一带 `[PREVIEW]` 前缀。

### 4. 重新部署 Vercel

迁移和 seed 完成后，重新部署 Vercel production / preview 环境，使应用使用最新 schema 和演示数据。

## 注意事项

- 必须设置 `PREVIEW_SEED=1` 才允许执行 `scripts/seed-preview-data.cjs`。
- 脚本启动时会输出脱敏后的 `DATABASE_URL`，执行前请确认目标库正确。
- 当前远程环境均为测试环境，可以清库重建业务测试数据。
- 不要使用 target-contract 的 reset 脚本直接操作远程库；target-contract 仍主要用于本地自动化验证。
- `PREVIEW_SEED_PASSWORD` 未设置时，测试用户默认密码为 `123456`。
